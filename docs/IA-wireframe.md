# SegmentFlow 화면 IA / Wireframe v1

## 1. 설계 목표

이 IA는 아래 3개 사용자 질문에 답하도록 설계한다.

1. **이 영상은 어떤 구조로 나뉘는가**
2. **어느 세그먼트에서 학습 흐름이 흔들리는가**
3. **광고를 어디에 넣는 것이 가장 안전한가**

핵심 UX 원칙은 다음과 같다.

* 운영자는 영상 전체가 아니라 **세그먼트 단위 흐름**을 봐야 한다.
* 추천 결과보다 먼저 **추천 이유와 위험도**를 이해해야 한다.
* 광고 성과와 학습 지표는 **같은 흐름 안에서 같이** 봐야 한다.
* 추천은 자동 집행이 아니라 **사람 승인 중심**이어야 한다.

# 2. 전체 IA 구조

```text
[로그인]

[메인 앱]
 ├─ 대시보드
 │   ├─ 전체 영상 현황
 │   ├─ 최근 분석 결과
 │   └─ 추천 대기 항목
 │
 ├─ 영상 관리
 │   ├─ 영상 목록
 │   ├─ 신규 영상 등록
 │   └─ 영상 상세
 │       ├─ 개요
 │       ├─ 세그먼트 검토
 │       ├─ 학습 퍼널 분석
 │       ├─ Agent 추천
 │       ├─ 성과 비교 리포트
 │       └─ 변경 이력 / 승인 로그
 │
 ├─ 추천 관리
 │   ├─ 추천 대기 목록
 │   ├─ 승인 완료 목록
 │   └─ 반려/수정 이력
 │
 ├─ 리포트
 │   ├─ 전체 성과 요약
 │   ├─ 영상별 성과 비교
 │   └─ 가드레일 위반 현황
 │
 └─ 설정
     ├─ 가드레일 KPI 설정
     ├─ 명시적 신호 설정
     └─ 조직/권한 설정
```

# 3. 전역 내비게이션 구조

## 좌측 사이드바

* 대시보드
* 영상 관리
* 추천 관리
* 리포트
* 설정

## 상단 글로벌 바

* 조직/워크스페이스 선택
* 검색
* 알림
* 사용자 프로필

## 상세 화면 내부 탭

영상 상세 화면에서는 다음 탭을 사용한다.

* 개요
* 세그먼트
* 학습 퍼널
* Agent 추천
* 성과 리포트
* 로그

이 구조가 좋은 이유는,
영상 하나에 대해 **등록 → 분석 → 추천 → 승인 → 검증**이 한 화면 맥락 안에서 이어지기 때문이다.

# 4. 핵심 화면 정의

## 4.1 로그인 화면

### 목적

운영자가 조직 계정으로 서비스에 진입한다.

### 주요 요소

* 로고
* 이메일 입력
* 비밀번호 입력 또는 SSO 로그인
* 에러 메시지
* 비밀번호 찾기

### 와이어프레임

```text
+--------------------------------------------------+
|                  SegmentFlow                     |
|      교육영상 학습 퍼널 기반 광고 추천 시스템         |
|                                                  |
|   [ 이메일 입력________________________ ]          |
|   [ 비밀번호 입력______________________ ]          |
|                                                  |
|   [ 로그인 ]                                     |
|   [ SSO로 로그인 ]                               |
|                                                  |
|   비밀번호를 잊으셨나요?                           |
+--------------------------------------------------+
```

---

## 4.2 대시보드

### 목적

운영자가 전체 운영 상태를 한눈에 파악하고, 우선 확인해야 할 영상을 찾는다.

### 이 화면에서 답해야 하는 질문

* 지금 분석이 완료된 영상은 몇 개인가
* 추천 대기 중인 건 무엇인가
* 가드레일 위반 위험이 있는 영상은 무엇인가

### 주요 구성

* 상단 KPI 카드
* 최근 업로드/최근 분석 완료 영상
* 추천 대기 리스트
* 위험 알림 카드
* 빠른 액션 버튼

### 주요 액션

* 신규 영상 등록
* 특정 영상 상세 진입
* 추천 대기 건 검토
* 리포트 진입

### 와이어프레임

```text
+----------------------------------------------------------------------------------+
| 로고 | 대시보드 | 영상 관리 | 추천 관리 | 리포트 | 설정              [검색] [프로필] |
+----------------------------------------------------------------------------------+

[ 전체 영상 128 ] [ 분석 완료 92 ] [ 추천 대기 14 ] [ 가드레일 경고 3 ]

+----------------------------------+  +------------------------------------------+
| 최근 분석 완료                   |  | 추천 대기 항목                           |
| - 영상 A / 분석 완료             |  | - 영상 B / 추천안 2개 / 승인 필요         |
| - 영상 C / 분석 완료             |  | - 영상 D / 추천안 1개 / 수정 필요         |
| - 영상 E / 분석 완료             |  | - 영상 F / 위험도 높음                    |
+----------------------------------+  +------------------------------------------+

+----------------------------------+  +------------------------------------------+
| 위험 알림                         |  | 빠른 시작                                 |
| - 영상 D: 다음 세그먼트 이동률 하락 |  | [ 신규 영상 등록 ]                        |
| - 영상 F: 이탈 급증               |  | [ 최근 추천 검토 ]                        |
+----------------------------------+  +------------------------------------------+
```

## 4.3 영상 목록 화면

### 목적

운영자가 전체 영상 자산을 관리하고 상태별로 필터링한다.

### 주요 컬럼

* 영상명
* 상태
* 세그먼트 수
* 최근 분석일
* 추천 상태
* 적용 여부
* 가드레일 상태

### 필터

* 분석 전 / 분석 중 / 분석 완료
* 추천 대기 / 승인 완료 / 반려
* 가드레일 경고 있음 / 없음

### 와이어프레임

```text
+----------------------------------------------------------------------------------+
| 영상 관리                                                          [신규 영상 등록] |
+----------------------------------------------------------------------------------+

[검색________________] [상태▼] [추천 상태▼] [가드레일▼]

+----------------------------------------------------------------------------------+
| 영상명        | 상태     | 세그먼트 | 최근 분석일 | 추천 상태 | 가드레일 | 상세       |
|----------------------------------------------------------------------------------|
| 영상 A        | 완료     | 12       | 03/10      | 승인 완료 | 정상      | [보기]     |
| 영상 B        | 완료     | 9        | 03/10      | 대기      | 정상      | [보기]     |
| 영상 C        | 분석 중   | -        | -          | -         | -         | [보기]     |
| 영상 D        | 완료     | 15       | 03/09      | 수정 필요 | 경고      | [보기]     |
+----------------------------------------------------------------------------------+
```

## 4.4 신규 영상 등록 화면

### 목적

새 교육영상을 분석 대상으로 등록한다.

### 주요 입력 항목

* 영상명
* 영상 파일 업로드 또는 URL
* 자막 업로드
* 카테고리/태그
* 기본 메타데이터
* 명시적 신호 설정 여부

  * 이해도 체크 사용 여부
  * 퀴즈/추가 반응 데이터 연동 여부

### 주요 액션

* 저장
* 업로드 후 분석 시작
* 임시 저장

### 와이어프레임

```text
+------------------------------------------------------------------+
| 신규 영상 등록                                                    |
+------------------------------------------------------------------+

영상명
[______________________________________________________________]

영상 파일
[ 파일 업로드 ] 또는 [ URL 입력________________________________ ]

자막 파일
[ SRT/VTT 업로드 ]   [ 자동 생성 사용 여부 □ ]

카테고리 / 태그
[__________________]  [__________________]

명시적 학습 신호 설정
[✓] 이해도 체크 사용
[ ] 퀴즈 연동
[ ] 핵심 개념 선택 연동
[ ] 한 줄 요약 연동

[ 임시 저장 ]                                  [ 등록 후 분석 시작 ]
```

# 5. 영상 상세 화면 구조

영상 상세는 이 제품의 핵심 화면이다.
하나의 화면 안에서 탭 기반으로 분석 흐름을 이어간다.

```text
영상 상세
 ├─ 개요
 ├─ 세그먼트
 ├─ 학습 퍼널
 ├─ Agent 추천
 ├─ 성과 리포트
 └─ 로그
```

---

## 5.1 영상 상세 - 개요 탭

### 목적

영상의 현재 상태를 한눈에 요약한다.

### 주요 정보

* 분석 상태
* 총 길이
* 세그먼트 수
* 최근 추천 상태
* 최근 적용 결과
* 위험 구간 요약

### 와이어프레임

```text
+----------------------------------------------------------------------------------+
| 영상 제목: 교육영상 A                                   [분석 재실행] [추천 검토] |
| 상태: 분석 완료 | 길이: 28분 | 세그먼트: 11 | 최근 추천: 대기                    |
+----------------------------------------------------------------------------------+

[ 완주율 72% ] [ 다음 세그먼트 이동률 65% ] [ 이해도 체크 응답률 24% ] [ CTR 3.8% ]

+----------------------------------+  +------------------------------------------+
| 위험 요약                         |  | 최근 추천 요약                           |
| - Segment 4 이탈 급증             |  | 추천 1: Segment 3 이후                   |
| - Segment 5 재시청 증가           |  | 추천 2: Segment 7 이후                   |
| - Segment 8 이해도 체크 저하      |  | 상태: 운영자 검토 대기                   |
+----------------------------------+  +------------------------------------------+

+----------------------------------------------------------------------------------+
| 타임라인 요약                                                               |
| [S1][S2][S3][S4!][S5↺][S6][S7][S8?][S9][S10][S11]                          |
+----------------------------------------------------------------------------------+
```

---

## 5.2 영상 상세 - 세그먼트 탭

### 목적

자동 생성된 세그먼트를 검토하고 필요 시 수정한다.

### 주요 정보

* 세그먼트 번호
* 시작/종료 시점
* 세그먼트 제목
* 핵심 개념
* 경계 수정 가능 여부

### 주요 액션

* 세그먼트 병합
* 세그먼트 분할
* 제목 수정
* 확정

### 와이어프레임

```text
+----------------------------------------------------------------------------------+
| [개요] [세그먼트] [학습 퍼널] [Agent 추천] [성과 리포트] [로그]                |
+----------------------------------------------------------------------------------+

+----------------------+-----------------------------------------------------------+
| 세그먼트 목록          | 세그먼트 상세                                              |
|----------------------|-----------------------------------------------------------|
| 1. 00:00~02:30      | 제목: 기본 개념 소개                                       |
| 2. 02:30~05:10      | 핵심 개념: A, B                                            |
| 3. 05:10~08:40      | 자막 미리보기                                              |
| 4. 08:40~11:20      | [텍스트 영역......................................]         |
| 5. 11:20~13:10      |                                                           |
| ...                 | [분할] [병합] [제목 수정] [확정]                           |
+----------------------+-----------------------------------------------------------+
```

---

## 5.3 영상 상세 - 학습 퍼널 탭

### 목적

세그먼트 단위 학습 흐름을 분석한다.

### 핵심 질문

* 어디서 이탈이 커지는가
* 어디서 되감기/재시청이 많아지는가
* 명시적 신호는 어디서 흔들리는가
* 다음 세그먼트 이동이 어디서 끊기는가

### 주요 구성

* 세그먼트 타임라인
* 퍼널 차트
* 세그먼트별 히트맵
* 명시적 신호 / 암묵적 신호 구분 뷰
* 위험 구간 카드

### 와이어프레임

```text
+----------------------------------------------------------------------------------+
| [개요] [세그먼트] [학습 퍼널] [Agent 추천] [성과 리포트] [로그]                |
+----------------------------------------------------------------------------------+

필터: [전체] [암묵적 신호] [명시적 신호] [가드레일 관점]

+----------------------------------------------------------------------------------+
| 타임라인                                                                          |
| S1 | S2 | S3 | S4 | S5 | S6 | S7 | S8 | S9 | S10 | S11                          |
|          정상   정상   이탈↑  재시청↑        정상   이해도↓                        |
+----------------------------------------------------------------------------------+

+----------------------------------+  +------------------------------------------+
| 세그먼트별 암묵적 신호 히트맵     |  | 세그먼트별 명시적 신호                    |
| completion                       |  | 이해도 체크 응답률                        |
| dropout                          |  | 이해함/애매함/다시보기 비율                |
| rewatch                          |  | (확장 시) 퀴즈/개념선택/요약               |
+----------------------------------+  +------------------------------------------+

+----------------------------------------------------------------------------------+
| 퍼널 테이블                                                                       |
| 세그먼트 | 진입률 | 완료율 | 재시청률 | 다음 세그먼트 이동률 | 이해도 체크 반응률 |
| S1       | 100%  | 93%   | 8%      | 90%                | 28%              |
| S2       | 90%   | 88%   | 10%     | 84%                | 24%              |
| S3       | 84%   | 80%   | 12%     | 76%                | 23%              |
| S4       | 76%   | 61%   | 29%     | 54%                | 15%              |
+----------------------------------------------------------------------------------+
```

## 5.4 영상 상세 - Agent 추천 탭

### 목적

광고 위치 추천안을 검토하고 승인 또는 수정한다.

### 핵심 질문

* 왜 이 지점이 추천되었는가
* 이 위치의 위험은 무엇인가
* 대안 위치는 무엇인가
* 학습 가드레일을 넘지 않는가

### 주요 구성

* 추천 위치 Top N
* 추천 근거 카드
* 위험도
* 기대 효과
* 대안 슬롯
* 승인/반려/수정

### 와이어프레임

```text
+----------------------------------------------------------------------------------+
| [개요] [세그먼트] [학습 퍼널] [Agent 추천] [성과 리포트] [로그]                |
+----------------------------------------------------------------------------------+

+----------------------------------------------------------------------------------+
| 추천 요약                                                                         |
| 추천안 #1: Segment 3 이후         신뢰도: 높음        위험도: 낮음               |
| 기대 효과: 광고 CTR 상승 가능 / 학습 흐름 훼손 낮음                               |
+----------------------------------------------------------------------------------+

+----------------------------------+  +------------------------------------------+
| 추천 근거                         |  | 제외/주의 근거                           |
| - Segment 3 종료가 개념 전환점    |  | Segment 4는 이탈 급증으로 제외            |
| - 직전 완료율 높음                |  | Segment 5는 재시청 많아 제외              |
| - 다음 세그먼트 이동률 안정        |  | Segment 8은 이해도 체크 저하              |
| - 명시적 신호 반응 안정            |  |                                          |
+----------------------------------+  +------------------------------------------+

+----------------------------------------------------------------------------------+
| 타임라인                                                                          |
| [S1][S2][S3] ★추천1 [S4][S5][S6][S7] ☆대안1 [S8][S9][S10][S11]                |
+----------------------------------------------------------------------------------+

+----------------------------------------------------------------------------------+
| 추천안 목록                                                                        |
| #1 S3 이후 | 이유 보기 | [승인] [수정] [반려]                                    |
| #2 S7 이후 | 이유 보기 | [승인] [수정] [반려]                                    |
+----------------------------------------------------------------------------------+
```

---

## 5.5 영상 상세 - 성과 리포트 탭

### 목적

추천 적용 후 광고 성과와 학습 지표를 함께 검증한다.

### 핵심 질문

* CTR은 개선되었는가
* 완주율/이동률/이해도 체크가 무너졌는가
* 추천안 간 차이는 무엇인가

### 주요 구성

* 적용 전/후 비교
* 비교군 대비 차이
* 가드레일 위반 여부
* 추천안별 결과

### 와이어프레임

```text
+----------------------------------------------------------------------------------+
| [개요] [세그먼트] [학습 퍼널] [Agent 추천] [성과 리포트] [로그]                |
+----------------------------------------------------------------------------------+

비교 기준: [적용 전/후] [A/B 테스트] [추천안별]

+----------------------------------+  +------------------------------------------+
| 광고 성과                         |  | 학습 가드레일                             |
| CTR: 3.1% → 3.8%                 |  | 완주율: 74% → 72%                        |
| 노출수: 10,200 → 10,050          |  | 다음 세그먼트 이동률: 67% → 65%          |
| 클릭수: 316 → 382                |  | 이해도 체크 긍정 반응: 62% → 61%         |
+----------------------------------+  +------------------------------------------+

+----------------------------------------------------------------------------------+
| 가드레일 상태: 정상                                                              |
| 허용 기준: 완주율 -3%p 이내 / 이동률 -3%p 이내 / 명시적 신호 급락 없음           |
+----------------------------------------------------------------------------------+

+----------------------------------------------------------------------------------+
| 추천안별 성과                                                                     |
| 추천안 | CTR 변화 | 완주율 변화 | 이동률 변화 | 이해도 체크 변화 | 상태            |
| #1     | +0.7%p   | -2.0%p      | -1.8%p      | -1.0%p           | 유지 가능       |
| #2     | +0.4%p   | -3.8%p      | -3.2%p      | -4.5%p           | 위험            |
+----------------------------------------------------------------------------------+
```

# 9. 화면 간 핵심 플로우

## 플로우 1. 신규 영상 등록부터 분석 완료까지

```text
대시보드
→ 신규 영상 등록
→ 영상 업로드 / 자막 연동
→ 분석 시작
→ 영상 상세 > 세그먼트 탭
→ 세그먼트 검토/확정
→ 학습 퍼널 탭 확인
```

## 플로우 2. 추천 검토 및 승인

```text
대시보드 또는 추천 관리
→ 영상 상세 > Agent 추천
→ 추천 근거 확인
→ 승인 / 수정 / 반려
→ 로그 저장
```

## 플로우 3. 적용 후 검증

```text
영상 상세 > 성과 리포트
→ CTR / 가드레일 비교
→ 추천안별 결과 검토
→ 필요 시 새 추천 재생성
```

# 10. MVP에서 반드시 필요한 화면

MVP에서는 아래 6개만 있어도 제품이 성립한다.

1. 로그인
2. 대시보드
3. 신규 영상 등록
4. 영상 상세 - 세그먼트 탭
5. 영상 상세 - 학습 퍼널 탭
6. 영상 상세 - Agent 추천 탭

성과 리포트와 추천 관리 화면은 MVP 직후 붙여도 되지만,
파일럿 운영까지 생각하면 **성과 리포트는 빠르게 포함하는 편이 좋다**.
