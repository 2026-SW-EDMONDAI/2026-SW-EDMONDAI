# 1. 과제 개요

## (1) 프로젝트 명

**SegmentFlow – 교육영상 세그먼트 기반 학습 퍼널 분석 및 Agent 광고 위치 추천 시스템**

## (2) 프로젝트 목적

교육영상 플랫폼에서 학습 품질 유지와 광고 수익 확보라는 두 목표를 하나의 의사결정 흐름 안에서 연결하는 운영 지원 시스템을 개발한다.

본 프로젝트는 교육영상을 **세그먼트 단위로 분석**하고, **명시적 학습 신호(이해도 체크 등 직접 반응)와 암묵적 학습 신호(시청 행동 기반)**를 통합 해석하여, **학습 흐름을 최대한 보존하는 광고 위치를 Agent가 추천**하는 웹 기반 운영 대시보드(SegmentFlow)를 개발하는 것을 목적으로 한다.

핵심 목표:
- 교육영상을 의미 기반 **세그먼트로 자동 분할**하고 운영자 검토 지원
- 세그먼트별 **학습 퍼널(진입→완료→이탈→재시청→전환→이해도 확인)** 분석
- 암묵적/명시적 학습 신호를 결합한 **학습 안정성 평가**
- AI Agent 기반 **설명 가능한 광고 위치 추천** 및 운영자 승인 플로우
- 추천 적용 후 **광고 성과 + 학습 가드레일** 동시 모니터링

# 2. 과제 수행 과정

## (1) 설계 및 계획

### 시스템 아키텍처

```text
[운영자 대시보드 - Next.js]
        ↓
      ALB
        ↓
[API 서버 - FastAPI]
   ├─ PostgreSQL (운영 DB)
   ├─ Amazon S3 (영상/자막/리포트)
   ├─ RabbitMQ (메시지 브로커)
   └─ OpenTelemetry (관측)
        ↓
[Celery Workers]
   ├─ video-processing (영상 분석)
   ├─ metrics-aggregation (퍼널 집계)
   └─ recommendation (Agent 추천 생성)
```

### DB 설계 (핵심 ERD)

```text
organizations
 └─ organization_members ─ users
 └─ videos
     ├─ video_assets
     ├─ caption_tracks → caption_cues
     ├─ video_signal_configs
     ├─ segment_sets → segments
     ├─ learner_sessions → learning_events
     ├─ segment_metric_snapshots
     ├─ recommendation_runs → recommendation_candidates
     │                         ├─ recommendation_evidence
     │                         └─ recommendation_reviews
     └─ ad_placements → placement_performance_daily
 └─ guardrail_policies
audit_logs
```

### 화면 IA 설계

```text
[메인 앱]
 ├─ 대시보드 (전체 현황, 최근 분석, 추천 대기 항목)
 ├─ 영상 관리
 │   ├─ 영상 목록 / 신규 등록
 │   └─ 영상 상세
 │       ├─ 개요 / 세그먼트 검토 / 학습 퍼널 분석
 │       ├─ Agent 추천 / 성과 비교 리포트
 │       └─ 변경 이력 / 승인 로그
 ├─ 추천 관리 (대기/승인/반려 목록)
 ├─ 리포트 (전체 성과, 영상별 비교, 가드레일 위반 현황)
 └─ 설정 (가드레일 KPI, 명시적 신호, 조직/권한)
```

## (2) 구현 내용

### M1. 프로젝트 골격 및 공통 기반 구축

- **모노레포 구조** 구성: `apps/web` (Next.js), `apps/api` (FastAPI), `apps/worker` (Celery), `packages/*` (공통 모듈)
- **Docker Compose** 개발환경: postgres, rabbitmq, api, worker, web 컨테이너 원커맨드 기동
- **FastAPI 기본 앱**: health check, version endpoint, 공통 exception handler, request id middleware, OpenAPI 문서
- **Next.js 레이아웃**: 로그인 화면, 좌측 사이드바, 상단 글로벌 바, 대시보드 shell
- **인증/권한**: JWT 인증, organization context, admin/operator/analyst role guard
- **DB 마이그레이션**: Alembic + 초기 핵심 테이블 (organizations, users, videos, guardrail_policies, audit_logs)
- **CI 파이프라인**: lint, type check, backend unit test, frontend build 자동화

### M2. 교육영상 등록 및 세그먼트 생성

- **영상 등록**: `POST /api/v1/orgs/{orgId}/videos` – 영상 파일 업로드, S3 저장, video_assets 메타데이터 기록
- **자막 연동**: caption_tracks / caption_cues 저장, 기본 자막 조회 API
- **분석 작업 트리거**: `POST /videos/{videoId}/analyze` – Celery job 비동기 실행, 상태 변경 (uploaded → processing → analyzed)
- **세그먼트 자동 생성**: 자막 기반 의미 단위 분절, 세그먼트별 title/topic 자동 생성, segment_set 버전 관리
- **세그먼트 검토**: 운영자 UI에서 세그먼트 목록 확인, 단건 수정/분할/병합, draft → finalized 상태 관리
- **감사 로그**: 영상 생성, 분석 시작, 세그먼트 수정, finalize 이력 자동 기록

### M3. 학습 퍼널 수집·집계·시각화

- **이벤트 수집 API**: `POST /api/v1/player/events/batch` – 플레이어 SDK에서 시청 이벤트 배치 수집
- **암묵적 신호 수집**: segment_start, segment_complete, pause, seek_back, rewatch, speed_change, next_segment_start, exit
- **명시적 신호 수집**: confidence_check_submit (understood / unsure / review_again)
- **메트릭 집계 워커**: raw events → segment_metric_snapshots (completion_rate, dropout_rate, rewatch_rate, next_transition_rate, explicit_response_rate, learning_stability_score)
- **위험 플래그 계산**: dropout spike, rewatch spike, low explicit signal, transition drop 자동 탐지
- **학습 퍼널 화면**: 타임라인 기반 세그먼트 시각화, 히트맵, 퍼널 테이블, explicit/implicit 필터 탭

### M4. Agent 추천 및 운영자 승인 플로우

- **추천 실행**: `POST /videos/{videoId}/recommendation-runs` – 비동기 추천 생성, 세그먼트 경계 후보 산출
- **추천 후보 생성**: 세그먼트 지표 기반 후보 생성 → 암묵적 신호 기반 학습 흐름 안정성 점검 → 명시적 신호 기반 이해도 저하 보정 → 가드레일 필터링 → LLM 기반 설명 생성 및 정렬
- **추천 근거(evidence)**: implicit 근거(완주율, 이탈률, 재시청률), explicit 근거(이해도 응답률), exclusion 근거(제외 사유), rationale summary
- **Agent 추천 화면**: Top N 추천, 추천 이유 카드(근거 지표/위험도/기대효과/confidence), 타임라인 위치 표시
- **운영자 승인 플로우**: approve / reject / modify / note – 최종 결정은 사람이 수행 (Human-in-the-loop)
- **placement 생성**: 승인/수정된 추천 → ad_placements 레코드 생성, 적용 상태 관리

### M5. 성과 리포트, 가드레일, 안정화

- **성과 집계**: ad_impression / ad_click 이벤트와 placement 연결, placement_performance_daily 일별 집계
- **비교 리포트**: 추천 적용 전후 CTR / completion_rate / transition_rate / explicit_signal 비교
- **가드레일 판정**: completion drop / transition drop / explicit signal drop threshold 기반 위반 판정
- **성과 리포트 화면**: 적용 전/후 비교 차트, 가드레일 상태 배지, candidate별 성과 비교
- **감사 로그 화면**: 추천 생성/승인/수정/적용/세그먼트 변경 이력 조회
- **E2E 테스트**: 핵심 happy path 자동화 (영상 등록 → 분석 → 세그먼트 수정 → 퍼널 → 추천 → 승인 → 리포트)

## (2) 기대효과 및 향후 계획

### 기대효과

1. **교육영상 운영 의사결정 고도화**: 세그먼트 단위 분석과 AI Agent 추천을 통해, 운영자가 "어디에 광고를 넣어야 학습을 보존하면서 성과를 낼 수 있는가"를 데이터 기반으로 판단할 수 있는 운영 도구 프로토타입을 확보하였다.

2. **다중 학습 신호 통합 분석 프레임워크 검증**: 암묵적 신호(시청 행동)와 명시적 신호(이해도 체크)를 통합 해석하는 접근이 퀴즈 단일 신호보다 학습 상태를 더 풍부하게 파악할 수 있음을 확인하였다.

3. **Human-in-the-loop AI 추천 시스템 실현**: 완전 자동화가 아닌 설명 가능한 추천 + 사람 승인 구조가 교육 도메인에서 운영자 신뢰를 확보하는 데 효과적임을 검증하였다.

4. **풀스택 프로덕션급 시스템 개발 경험**: 프론트엔드(Next.js), 백엔드(FastAPI), 비동기 파이프라인(Celery/RabbitMQ), AI(LLM Agent), 인프라(Docker/AWS)를 아우르는 실무 수준 개발 역량을 축적하였다.
