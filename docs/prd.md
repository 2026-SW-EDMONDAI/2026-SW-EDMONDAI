# PRD v1.1

## SegmentFlow (가칭)

**교육영상 세그먼트 기반 학습 퍼널 분석 및 Agent 광고 위치 추천 시스템**

본 제품은 교육영상을 업로드하면 **세그먼트를 생성하고**, 세그먼트별 학습 퍼널을 분석한 뒤, **학습 흐름을 최대한 해치지 않는 광고 위치를 Agent가 추천**하는 운영 도구를 목표로 한다.

이때 학습 퍼널은 단순 시청 데이터만이 아니라,
**암묵적 학습 신호(시청 행동 기반)**와 **명시적 학습 신호(직접 반응 기반)**를 함께 활용해 해석한다.

**MVP 포함 범위**

* 교육영상 업로드 및 자막 연동
* 세그먼트 자동 생성
* 세그먼트 단위 학습 퍼널 지표 산출
* 암묵적 신호 수집 및 분석
* 명시적 신호 중 `이해도 체크` 수집 및 분석
* Agent 기반 광고 위치 추천
* 추천 근거 확인 및 운영자 승인

**MVP 제외 범위**

* 개인별 맞춤 광고 노출
* 실시간 광고 입찰 최적화
* 광고 크리에이티브 자동 생성
* 운영자 승인 없는 자동 광고 삽입
* 고비용 자유서술형 평가 자동 채점의 전면 도입

# 1. 제품 개요 (Product Overview)

## 1.1 제품명

* **공식 가칭:** SegmentFlow
* **영문 설명명:** Segment-level Learning Funnel Intelligence for Educational Videos

## 1.2 제품 비전

교육영상의 수익화가 학습 몰입을 깨뜨리는 것이 아니라, **학습 흐름을 이해한 상태에서 가장 덜 방해되는 방식으로 운영되도록 돕는 것**이 제품의 비전이다.

이 제품은 교육영상 운영 환경에서 흔히 분리되어 다뤄지는 두 목표, 즉 **학습 품질 유지**와 **광고 수익 확보**를 하나의 의사결정 흐름 안에서 연결한다. 궁극적으로는 “광고를 넣을 수 있는가”가 아니라, **“어디에 넣어야 학습을 최대한 보존하면서도 운영 성과를 낼 수 있는가”**를 판단하는 운영 표준을 만드는 것이 목표다.

## 1.3 서비스 정의

SegmentFlow는 교육영상 플랫폼 운영자와 콘텐츠 기획자를 위한 **웹 기반 운영 대시보드 + 추천 엔진**이다.

핵심 서비스는 다음 세 단계로 구성된다.

1. 업로드된 교육영상을 **세그먼트**로 분할한다.
2. 세그먼트별 시청 행동 데이터와 학습 반응 데이터를 기반으로 **학습 퍼널**을 분석한다.
3. 분석 결과를 바탕으로 Agent가 **광고 위치 후보와 그 이유를 제안**한다.

즉, 이 제품은 광고 집행 도구가 아니라, **교육 맥락과 학습 반응을 반영한 광고 의사결정 지원 도구**다.

## 1.4 서비스 가치 제안 (Value Proposition)

| 관점        | 제공 가치                                                                                   |
| --------- | --------------------------------------------------------------------------------------- |
| 학습자       | 개념 설명 중간이 아닌 자연스러운 전환 지점 중심으로 광고가 배치되어 학습 흐름 훼손을 줄인다.                                   |
| 운영자       | 영상 단위 평균값이 아니라 세그먼트 단위로 학습 이탈, 반복 시청, 이해도 반응 구간을 파악하고 설명 가능한 추천을 받아 운영 결정을 빠르게 내릴 수 있다. |
| 콘텐츠 기획자   | 어떤 구간이 학습상 민감한지, 어떤 구간에서 이해도 저하가 발생하는지 확인해 콘텐츠 구조 자체를 개선할 수 있다.                         |
| 광고/수익 운영자 | 무작정 광고 수를 늘리기보다, 상대적으로 안전한 슬롯 후보를 기반으로 성과 실험을 설계할 수 있다.                                 |
| 플랫폼 생태계   | 학습 품질을 해치지 않는 광고 운영 원칙을 축적하여 장기적으로 신뢰 가능한 교육영상 수익 모델을 만든다.                              |

# 2. 개발 배경 및 필요성 (Background)

## 2.1 문제 정의

### 문제 1. 기존 분석은 영상 단위 평균값에 머무는 경우가 많다

교육영상 운영에서는 완주율, 평균 시청시간, 평균 시청률 같은 **영상 단위 지표**가 자주 사용된다. 그러나 이런 지표만으로는 다음을 파악하기 어렵다.

* 어느 구간에서 학습자가 이탈하는지
* 어떤 개념 전환 지점이 부담이 큰지
* 광고 삽입이 특히 위험한 구간이 어디인지
* 특정 세그먼트에서 이해도 저하가 있었는지

즉, 영상 전체가 아니라 **세그먼트 단위 흐름**을 보는 해상도가 필요하다.

### 문제 2. 광고 운영과 학습 품질 관리가 분리되어 있다

교육 플랫폼에서는 보통 학습 분석과 수익 운영이 다른 기준으로 움직인다. 그 결과 다음 문제가 발생한다.

* 광고 위치가 학습 문맥보다 운영 편의에 의해 정해짐
* 광고 성과와 학습 성과의 상호작용이 충분히 검토되지 않음
* “수익이 올랐지만 학습 경험은 나빠진” 상태를 뒤늦게 발견하게 됨

### 문제 3. 기존 최적화는 단일 목적에 치우치기 쉽다

많은 광고 최적화 로직은 ROI, CTR, ROAS 같은 수익 지표 중심으로 설계된다. 그러나 교육영상에서는 광고 성과가 좋아도 다음이 함께 나빠질 수 있다.

* 세그먼트 완주율
* 재시청률의 비정상적 증가
* 이후 구간 진입률
* 이해도 체크 반응
* 요약/핵심 개념 선택과 같은 학습 확인 신호

교육영상에서는 수익 지표만으로 “좋은 위치”를 판단하기 어렵다.

### 문제 4. 운영자는 결과보다 이유가 필요하다

운영자는 추천을 그대로 따르기보다, 왜 해당 위치가 추천되었는지 알아야 한다. 특히 교육 콘텐츠에서는 다음 질문이 중요하다.

* 이 지점이 개념 전환 구간인가
* 이 지점은 이탈 위험이 낮은가
* 직전 세그먼트의 학습 이해 신호가 안정적인가
* 광고를 넣어도 학습 흐름을 크게 깨지 않는가

따라서 **설명 가능한 추천**이 필요하다.

### 문제 5. 퀴즈만으로 학습 상태를 대표하기 어렵다

교육영상 환경에서는 항상 퀴즈가 존재하는 것이 아니며, 퀴즈만으로 실제 학습 상태를 충분히 설명하기도 어렵다.
실제 학습 반응은 다음과 같이 다양하게 나타날 수 있다.

* 반복 시청과 되감기
* 이해도 체크
* 핵심 개념 선택
* 간단한 요약 입력
* 다음 세그먼트 자연 전환

따라서 **학습 품질 평가는 퀴즈 단일 신호가 아니라 명시적 신호와 암묵적 신호를 함께 보는 구조**가 필요하다.

## 2.2 기회 영역

### 기술적 기회

* 영상 자막, 시청 이벤트, 명시적 학습 반응 데이터를 연결하면 세그먼트 단위 분석이 가능해졌다.
* LLM/Agent 구조는 단일 점수 계산보다 **복수 신호를 종합해 이유를 제시하는 의사결정 보조**에 적합하다.
* 운영툴 관점에서는 완전 자동화보다 **Human-in-the-loop 추천형 시스템**이 도입 장벽이 낮다.

### 시장적 기회

* 교육영상 자산은 계속 늘고 있지만, 수익화와 학습 품질 관리가 함께 설계된 도구는 드물다.
* 플랫폼 입장에서는 광고를 없애기보다, **덜 해로운 방식으로 운영하는 도구**가 현실적이다.
* 콘텐츠 팀은 단순 성과 대시보드가 아니라, **실행 가능한 개선 제안**을 원한다.

## 2.3 핵심 개념 정의

| 용어        | 정의                                                                           |
| --------- | ---------------------------------------------------------------------------- |
| 교육영상      | 학습 목적이 있는 영상 콘텐츠. 강의, 튜토리얼, 설명형 콘텐츠, 기관 교육영상 등을 포함한다.                        |
| 세그먼트      | 교육영상을 의미와 개념 흐름 기준으로 나눈 최소 분석 단위.                                            |
| 학습 퍼널     | 세그먼트 진입 → 세그먼트 시청 완료 → 재시청/이탈 → 다음 세그먼트 이동 → 이해도 확인 신호 발생으로 이어지는 학습 흐름.      |
| 학습 신호     | 세그먼트별 학습 상태를 판단하기 위해 수집하는 반응 데이터의 총칭. 명시적 신호와 암묵적 신호로 구분한다.                  |
| 명시적 학습 신호 | 학습자가 직접 입력하거나 선택하는 반응 데이터. 예: 이해도 체크, 핵심 개념 선택, 한 줄 요약, 체크포인트 응답             |
| 암묵적 학습 신호 | 별도 입력 없이 행동 로그로 관찰되는 반응 데이터. 예: 완료율, 이탈률, 재시청률, 되감기, 일시정지, 배속 변화, 다음 세그먼트 이동 |
| 광고 위치 후보  | 광고 삽입이 가능한 시점 중, 학습 흐름 방해 위험이 상대적으로 낮다고 판단된 구간                               |
| Agent 추천  | 세그먼트 지표, 문맥, 운영 제약을 종합해 광고 위치와 사유를 제안하는 의사결정 지원 결과                           |
| 운영자 승인    | 추천 결과를 사람이 검토하고 적용 여부를 최종 결정하는 절차                                            |
| 가드레일 KPI  | 수익 성과를 추구하더라도 일정 수준 이상 악화되면 안 되는 학습 지표                                       |

# 3. 제품 목표 및 KPI (Goals & Metrics)

## 3.1 비즈니스 목표

### 1단계: 문제-해결 적합성 검증

* 세그먼트 단위 분석이 영상 단위 분석보다 운영 의사결정에 유의미한지 검증
* 명시적 신호와 암묵적 신호를 함께 사용할 때 추천 품질이 개선되는지 확인
* 운영자가 추천 결과를 실제 업무에 활용할 수 있는지 확인

### 2단계: 파일럿 운영 정착

* 운영팀이 신규 교육영상 분석 시 SegmentFlow를 기본 도구로 사용하도록 정착
* 광고 위치 결정 과정에서 “직관”보다 “데이터 + 추천 근거” 기반 검토 비중 확대

### 3단계: 성과 검증

* 추천 위치 적용군에서 학습 가드레일을 유지하면서 광고 성과 개선 가능성 확인
* 추천 결과의 승인율, 재사용률, 반복 방문률 확보

## 3.2 핵심 KPI

### 운영 활성 지표

| KPI        | 정의                             |
| ---------- | ------------------------------ |
| 운영자 WAU    | 주 1회 이상 분석 또는 추천 검토를 수행한 운영자 수 |
| 분석 완료 영상 수 | 세그먼트 생성과 퍼널 분석이 완료된 영상 수       |
| 반복 사용률     | 첫 사용 후 다음 주에도 재사용한 운영자 비율      |

### 데이터/분석 지표

| KPI         | 정의                              |
| ----------- | ------------------------------- |
| 세그먼트 생성 성공률 | 업로드 영상 중 정상적으로 세그먼트 생성까지 완료된 비율 |
| 분석 소요시간     | 업로드 후 추천 결과가 생성될 때까지 걸리는 시간     |
| 이벤트 수집 완전성  | 필수 시청 이벤트가 누락 없이 적재된 비율         |
| 명시적 신호 응답률  | 이해도 체크 등 직접 반응형 이벤트에 응답한 비율     |
| 암묵적 신호 커버리지 | 세그먼트별 필수 행동 로그가 정상 수집된 비율       |

### 추천 성과 지표

| KPI       | 정의                            |
| --------- | ----------------------------- |
| 추천 승인율    | 운영자가 제안된 광고 위치를 승인한 비율        |
| 추천 수정률    | 추천 후 운영자가 위치를 변경한 비율          |
| 추천 설명 만족도 | 운영자가 추천 이유를 이해 가능하다고 평가한 비율   |
| 가드레일 위반률  | 추천 적용 후 학습 KPI가 허용 하락폭을 넘긴 비율 |

### 비즈니스/매출 지표

| KPI        | 정의                                              |
| ---------- | ----------------------------------------------- |
| 광고 CTR 개선율 | 추천 위치 적용군의 CTR이 비교군 대비 얼마나 개선되었는지               |
| 광고 노출 유지율  | 추천 적용 후 정상적으로 확보된 광고 노출 비율                      |
| 학습 품질 보존율  | 적용 후 완주율, 다음 세그먼트 이동률, 이해도 체크 반응이 기준선 대비 유지된 수준 |

# 4. 핵심 사용자 정의 (User Personas)

## Persona A. 김지훈 / 교육 플랫폼 운영자 / 데이터 중심 실무형

* **역할:** 플랫폼 운영 총괄, 콘텐츠 성과와 운영 효율 관리
* **니즈:** 어떤 영상에 광고를 넣어도 되는지 빠르게 판단하고 싶다.
* **페인포인트:** 지금은 평균 지표와 감으로 판단해서 일관성이 떨어진다.
* **성공 기준:** 추천 결과가 빠르고, 이유가 명확하며, 운영 승인에 필요한 정보가 한 화면에 보인다.

## Persona B. 박서연 / 콘텐츠 기획자 / 학습 경험 민감형

* **역할:** 교육영상 구조 설계, 콘텐츠 개선
* **니즈:** 어느 구간이 학습적으로 민감한지 알고 싶다.
* **페인포인트:** 광고가 학습 흐름을 어디서 끊는지 구조적으로 파악하기 어렵고, 퀴즈가 없는 구간은 이해도를 보기 힘들다.
* **성공 기준:** 세그먼트별 이탈, 재시청, 이해도 체크, 요약/반응 흐름을 보고 “광고를 넣으면 안 되는 구간”을 설명할 수 있다.

## Persona C. 이민호 / 광고 운영 매니저 / 성과 책임형

* **역할:** 광고 슬롯 운영, 성과 실험 설계
* **니즈:** 교육 콘텐츠 특성을 해치지 않으면서도 광고 성과가 나는 위치를 찾고 싶다.
* **페인포인트:** 무조건 중간에 넣으면 반응이 나빠지고, 무조건 뒤로 미루면 성과가 떨어질 수 있다.
* **성공 기준:** 추천 슬롯을 기반으로 실험군/비교군을 설계하고, 성과 차이를 쉽게 확인할 수 있다.

### 간접 사용자: 학습자

학습자는 대시보드를 직접 사용하지 않지만, 본 제품이 최종적으로 보호해야 하는 대상이다.
따라서 제품 설계의 기본 원칙은 **“학습자 경험 보존”**이다.

# 5. 사용자 여정 (User Flow)

## 5.1 End-to-End 여정

### 1단계. 운영자 최초 진입

운영자는 대시보드에 로그인하고 영상 분석 프로젝트를 생성한다.

### 2단계. 교육영상 등록

영상 파일과 자막을 업로드하거나 기존 영상 URL/ID를 연결한다.

### 3단계. 세그먼트 분석

시스템은 자막과 영상 메타데이터를 활용해 세그먼트를 생성하고, 운영자는 이를 검토한다.

### 4단계. 학습 퍼널 집계

플레이어에서 수집된 **시청 이벤트(암묵적 신호)**와 **이해도 체크 등 직접 반응 데이터(명시적 신호)**를 세그먼트 기준으로 집계한다.

### 5단계. Agent 추천 검토

Agent는 광고 위치 후보, 추천 근거, 위험도, 기대 성과를 제시한다.

### 6단계. 승인 및 반영

운영자는 추천을 승인하거나 수정하고, 적용 대상을 지정한다.

### 7단계. 성과 모니터링

적용 후 운영자는 CTR, 완주율, 다음 세그먼트 이동률, 이해도 체크 반응 등 가드레일 포함 결과를 확인한다.

### 8단계. 재방문

운영자는 신규 영상이나 성과 이상 징후가 있는 기존 영상에 대해 동일 흐름을 반복한다.

## 5.2 주요 흐름 다이어그램

### 흐름 A. 신규 영상 분석

```text
로그인
→ 프로젝트 생성
→ 영상/자막 업로드
→ 세그먼트 자동 생성
→ 세그먼트 검토/확정
→ 학습 퍼널 분석 결과 확인
```

### 흐름 B. 광고 위치 추천 검토

```text
세그먼트 퍼널 화면 진입
→ Agent 추천 결과 확인
→ 추천 이유/위험도 확인
→ 승인 또는 위치 수정
→ 추천안 저장
```

### 흐름 C. 적용 후 성과 모니터링

```text
추천안 적용
→ 광고/학습 성과 수집
→ 비교군 대비 변화 확인
→ 가드레일 위반 여부 확인
→ 다음 운영안 재조정
```

## 6.2 제품이 제공해야 하는 핵심 경험

* 평균값이 아닌 **구간별 흐름을 본다**
* 퀴즈 유무와 관계없이 **명시적 신호 + 암묵적 신호를 함께 본다**
* 추천 결과뿐 아니라 **추천 이유를 본다**
* 수익 지표뿐 아니라 **학습 가드레일을 함께 본다**
* 자동화보다 **승인 가능한 추천**을 제공한다

# 9. 시스템 아키텍처 및 기술 스택

## 9.1 상위 아키텍처

```text
[운영자 대시보드]
        ↓
[API 서버 / 인증]
        ↓
[영상/자막 전처리]
        ↓
[세그먼트 생성 엔진]
        ↓
[세그먼트 학습 퍼널 분석 엔진]
        ↓
[학습 신호 통합 계층]
        ↓
[Agent 추천 엔진]
        ↓
[추천 결과 저장 / 대시보드 노출]
        ↓
[승인 결과 Export/API 연동]
```

별도로 플레이어 측에서는 다음 데이터가 수집된다.

```text
[비디오 플레이어 SDK]
→ view_start
→ segment_start
→ segment_complete
→ pause
→ seek_back
→ rewatch
→ speed_change
→ subtitle_toggle
→ next_segment_start
→ confidence_check_submit
→ quiz_submit (optional)
→ concept_select (optional)
→ summary_submit (optional)
→ ad_impression
→ ad_click
```

## 9.2 컴포넌트 설명

### 1. 콘텐츠 입력 계층

* 영상 파일 업로드
* 자막 파일 업로드 또는 기존 자막 연결
* 콘텐츠 메타데이터 저장

### 2. 세그먼트 생성 엔진

* 자막 타임코드 정렬
* 의미 단위 분리
* 세그먼트 제목/주제 생성
* 세그먼트 경계 후보 산출

### 3. 학습 퍼널 분석 엔진

* 세그먼트 진입률, 완료율, 이탈률 계산
* 재시청 패턴 및 seek-back 분석
* 일시정지, 배속 변화, 자막 사용 등 암묵적 신호 집계
* 이해도 체크, 퀴즈, 요약 등 명시적 신호 연결
* 위험 구간 스코어링

### 4. 학습 신호 통합 계층

* 암묵적 신호와 명시적 신호 정규화
* 세그먼트별 학습 안정성 점수 계산
* 신호 부족 구간과 cold-start 상태 표기

### 5. Agent 추천 엔진

* 후보 위치 스코어링
* 가드레일 기반 제외 조건 적용
* 추천 우선순위 정렬
* 추천 이유 및 confidence 생성

### 6. 운영 대시보드

* 타임라인 기반 세그먼트 시각화
* 퍼널/히트맵/비교 리포트 제공
* 추천 승인/수정/저장

## 9.3 권장 기술 스택

| 영역          | 권장 스택                            | 역할                  |
| ----------- | -------------------------------- | ------------------- |
| Frontend    | React 또는 Next.js                 | 운영 대시보드 UI          |
| Backend API | Python FastAPI                   | 분석/추천 API 제공        |
| 비동기 처리      | Celery / Queue 기반 워커             | 영상 분석 및 배치 작업       |
| 데이터베이스      | PostgreSQL                       | 메타데이터, 추천 결과, 승인 로그 |
| 분석 저장소      | 데이터 웨어하우스 계열                     | 이벤트 로그 집계           |
| 파일 저장소      | Object Storage                   | 영상, 자막, 리포트 파일 저장   |
| AI/NLP      | Python 기반 NLP 파이프라인 + LLM API/모델 | 세그먼트 생성, 설명 생성      |
| 시각화         | Charting library                 | 타임라인/퍼널/히트맵 렌더링     |
| 인증          | SSO 또는 이메일 기반 조직 계정              | 운영자 접근 제어           |

### 기술 원칙

* 모델보다 **데이터 스키마와 운영 플로우**를 우선한다.
* 추천은 black box가 아니라 **설명 가능한 hybrid logic**를 지향한다.
* 대규모 실시간 시스템보다 **파일럿 가능한 비동기 분석 시스템**으로 시작한다.
* 학습 평가는 퀴즈 단일 신호가 아니라 **명시적/암묵적 신호 결합 구조**를 사용한다.

## 10.2 데이터 품질 규칙

* 세그먼트 타임코드 겹침 금지
* 이벤트 타임스탬프 역전 금지
* 익명 사용자 ID 일관성 유지
* 1개 영상 내 세그먼트 ID 중복 금지
* 명시적 신호는 세그먼트 ID와 반드시 연결되어야 함
* 명시적/암묵적 신호의 이벤트 타입은 명확히 구분 저장해야 함

## 10.3 추천 검증 전략

추천 정확도는 단일 정답이 아니라 운영 실효성으로 평가한다.

### 검증 방식

* 기존 운영 위치 대비 비교
* 운영자 승인율 측정
* 명시적 신호 유무별 추천 품질 비교
* 적용 후 학습 가드레일 유지 여부 확인
* 소규모 A/B 테스트 또는 전후 비교

### 핵심 검증 질문

* 추천 위치는 운영자가 실제로 “쓸 만하다”고 느끼는가
* 추천 적용 후 CTR 개선이 있는가
* 학습 KPI 하락이 허용 범위 안에 있는가
* 명시적 신호가 추가될 때 추천 설명력과 신뢰도가 높아지는가

## 10.4 주요 리스크와 대응

| 리스크          | 설명                                  | 대응 방안                                 |
| ------------ | ----------------------------------- | ------------------------------------- |
| 잘못된 세그먼트 분할  | 의미 단위가 부정확하면 이후 분석 전체가 흔들림          | 세그먼트 검토/수정 기능 제공, 규칙 기반 fallback 적용   |
| 데이터 부족       | 신규 영상은 시청 이벤트가 적어 추천 신뢰도가 낮음        | cold-start 경고, 문맥 기반 추천 우선 사용         |
| 명시적 신호 응답 저조 | 이해도 체크 등 직접 반응 데이터가 충분히 쌓이지 않을 수 있음 | 1탭형 UI, 일부 세그먼트 샘플링, 보조 신호로 암묵적 신호 활용 |
| 사용자 피로도 증가   | 명시적 입력이 많으면 학습 흐름을 해칠 수 있음          | MVP는 이해도 체크 1종 중심, 나머지는 선택적/간헐적 노출    |
| 광고 수익 편향     | 수익 신호만 강하면 학습 KPI 훼손 가능             | 가드레일 KPI 필수 적용                        |
| 설명 불신        | 추천 이유가 모호하면 운영자가 사용하지 않음            | 추천 근거 수치, 위험도, 제외 이유 함께 표기            |
| 법적/윤리적 이슈    | 미성년 학습자 데이터 등 민감성 존재                | 최소 수집, 익명화, 접근권한 통제                   |

# 11. UI/UX 설계 방향

## 11.1 설계 원칙

### 1. 타임라인 중심

운영자가 가장 먼저 이해해야 하는 것은 “영상 전체 흐름 속 어느 위치인가”다.
따라서 세그먼트, 이탈, 재시청, 추천 슬롯은 모두 **하나의 타임라인 상에서 연결**되어 보여야 한다.

### 2. 명시적/암묵적 신호의 구분 가시화

운영자는 단순히 점수만 보는 것이 아니라, **이 결과가 행동 기반인지 직접 반응 기반인지** 구분해서 볼 수 있어야 한다.

### 3. 설명 우선

추천 결과는 점수만 보여주지 않는다. 왜 추천되었는지, 왜 제외되었는지, 어떤 KPI를 참고했는지 함께 보여준다.

### 4. 오버라이드 친화적

추천을 따르지 않아도 불편하지 않아야 한다. 운영자가 위치를 바꾸면 시스템은 그 변경을 기록하고 이후 학습에 활용할 수 있어야 한다.

### 5. 가드레일 가시화

광고 성과 차트와 학습 지표 차트를 분리하지 않고 함께 본다. “성과는 올랐지만 학습은 깨졌다”를 쉽게 발견할 수 있어야 한다.

## 11.2 핵심 화면

### 1. 영상 분석 상세 화면

* 세그먼트 타임라인
* 세그먼트별 completion/dropout/rewatch 히트맵
* 명시적 신호 반응률 표시
* 위험 구간 표시

### 2. Agent 추천 화면

* 추천 광고 위치 Top N
* 추천 이유 카드
* 암묵적 신호 근거 / 명시적 신호 근거 구분 표시
* 기대효과 / 위험도 / confidence
* 승인 / 수정 / 제외 버튼

### 3. 성과 비교 리포트 화면

* 적용 전후 비교
* 추천안별 결과
* 가드레일 위반 여부
* 운영 메모/판단 근거

## 11.3 시각화 요소

* 타임라인 막대
* 세그먼트별 히트맵
* 전환 퍼널
* 명시적 신호 응답 배지
* 전후 비교 카드
* 추천 이유 태그

# 12. 제품 로드맵 (Roadmap)

## Phase 0. 요구사항 정제 및 데이터 준비 (0~4주)

**목표:** MVP 구현에 필요한 데이터 스키마와 운영 기준 확정

주요 산출물

* 이벤트 정의서
* 세그먼트 생성 기준서
* 명시적/암묵적 신호 정의서
* 가드레일 KPI 정의
* 화면 IA 초안

완료 기준

* 최소 이벤트 스키마 확정
* 세그먼트/추천 데이터 모델 확정
* MVP 명시적 신호 유형 확정 (`이해도 체크`)

## Phase 1. MVP 구축 (5~10주)

**목표:** 교육영상 분석부터 추천 생성까지의 기본 흐름 완성

주요 기능

* 영상/자막 업로드
* 세그먼트 자동 생성
* 암묵적 신호 기반 퍼널 계산
* 이해도 체크 기반 명시적 신호 수집
* Agent 추천 결과 생성
* 추천 근거 노출
* 승인/수정 기능

완료 기준

* 운영자가 한 편의 영상을 등록하고 추천안을 확인할 수 있음
* 추천 결과가 저장되고 재조회 가능
* 명시적/암묵적 신호가 구분되어 리포트에 표시됨

## Phase 2. 파일럿 운영 (11~18주)

**목표:** 실제 운영 환경에서 추천 수용성과 유용성 검증

주요 기능

* 비교 리포트
* 승인 버전 관리
* 추천 품질 모니터링
* 기본 Export/API 연동
* 퀴즈 또는 핵심 개념 선택 연동

완료 기준

* 실제 파일럿 콘텐츠 셋에서 운영자 승인율 측정 가능
* 추천 적용 전후 비교 리포트 제공 가능
* 명시적 신호 추가 시 추천 신뢰도 변화를 검증 가능

## Phase 3. 성과 최적화 및 안정화 (19~24주)

**목표:** 추천 품질 고도화와 운영 정착

주요 기능

* 추천 로직 개선
* cold-start 처리
* 신뢰도/위험도 개선
* 운영 메모 및 학습 데이터 축적
* 한 줄 요약 등 확장형 명시적 신호 도입 검토

완료 기준

* 6개월 KPI 중 핵심 항목 측정 가능
* 반복 사용 가능한 운영 도구 수준 안정성 확보
