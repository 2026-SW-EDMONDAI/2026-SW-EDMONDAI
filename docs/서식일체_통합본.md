# [붙임 제3호 서식] 현장미러형 연계 프로젝트 수행 계획서

## 1. 과제 개요

### (1) 프로젝트 명

SegmentFlow – 교육영상 세그먼트 기반 학습 퍼널 분석 및 Agent 광고 위치 추천 시스템

### (2) 프로젝트 목적

**배경 및 문제 인식**

- 현재 교육영상 플랫폼에서는 완주율·평균 시청시간 등 **영상 전체 단위 평균 지표**만 활용하는 경우가 많음.
- 영상 전체 평균값만으로는 어느 구간에서 학습자가 이탈하는지, 어떤 개념 전환 지점이 학습 부담이 큰지, 광고 삽입이 특히 위험한 구간이 어디인지 파악하기 어려움.
- 광고 위치 결정이 학습 문맥보다 **운영 편의 중심**으로 이루어져, "수익은 올랐지만 학습 경험이 나빠진" 상태를 사후에야 인지하는 문제가 반복됨.
- 기존 광고 최적화 로직은 CTR·ROAS 등 **수익 지표 단일 목적**에 치우쳐 있어, 교육영상 특유의 학습 품질 저하 리스크를 반영하지 못함.
- 퀴즈가 없는 구간에서는 학습 상태를 파악할 수단이 부족하며, 퀴즈만으로는 실제 학습 반응을 충분히 설명하기 어려움.
- 운영자는 추천 결과를 단순히 따르기보다 **"왜 이 위치가 추천되었는지"**를 이해해야 교육적 맥락에서 의사결정을 내릴 수 있으나, 기존 도구들은 이를 제공하지 못함.

**프로젝트 목적**

- 교육영상을 자막/문맥 기반으로 **세그먼트(의미 단위 구간)로 자동 분할**하고, 운영자가 이를 검토·수정할 수 있도록 함.
- 세그먼트별로 **학습 퍼널(진입 → 완료 → 이탈 → 재시청 → 다음 세그먼트 전환 → 이해도 확인)**을 분석하여, 영상 전체가 아닌 구간 단위로 학습 흐름을 파악함.
- **암묵적 학습 신호**(완료율, 이탈률, 되감기, 일시정지, 배속 변화 등 시청 행동 기반)와 **명시적 학습 신호**(이해도 체크, 핵심 개념 선택 등 직접 반응 기반)를 함께 수집·통합 분석하여, 퀴즈 유무에 관계없이 세그먼트별 학습 안정성을 판단함.
- 분석 결과를 기반으로 **AI Agent가 광고 위치 후보를 산출**하고, 각 후보에 대해 **추천 근거(근거 지표, 위험도, 기대효과)**를 설명 가능하게 제시함.
- 최종 광고 위치 결정은 자동 집행이 아닌 **운영자의 승인·수정·반려(Human-in-the-loop)**를 거치도록 설계함.
- 추천 적용 이후에는 광고 성과(CTR 등)뿐 아니라 **학습 가드레일 KPI(세그먼트 완주율 하락폭, 다음 세그먼트 이동률 변화 등)**를 동시에 모니터링하여, 학습 품질을 훼손하지 않는 범위에서 수익 성과를 검증함.

**핵심 가치 제안**

- **학습자** 관점: 개념 설명 도중이 아닌 자연스러운 전환 지점 중심으로 광고가 배치되어 학습 흐름 훼손을 줄임.
- **운영자** 관점: 세그먼트 단위로 학습 이탈·반복 시청·이해도 반응 구간을 파악하고, 설명 가능한 추천을 받아 운영 결정을 빠르게 내릴 수 있음.
- **콘텐츠 기획자** 관점: 어떤 구간이 학습상 민감한지, 이해도 저하가 발생하는 구간이 어디인지 확인하여 콘텐츠 구조 개선에 활용할 수 있음.
- **플랫폼 생태계** 관점: 학습 품질을 해치지 않는 광고 운영 원칙을 데이터로 축적하여 장기적으로 신뢰 가능한 교육영상 수익 모델을 구축함.

## 2. 수행 계획

### (1) 요구사항 분석

**핵심 기능 요구사항**

| ID | 요구사항 | 설명 | 우선순위 |
|----|---------|------|---------|
| FR-01 | 영상 업로드 | - 운영자는 교육영상 파일을 업로드하여 시스템에 등록할 수 있어야 함. | Must |
| FR-02 | 자막 연동 | - 자막 파일(SRT/VTT 등)을 업로드하거나, 기존 자막 데이터를 연결할 수 있어야 함. | Must |
| FR-03 | 세그먼트 자동 생성 | - 시스템은 자막 타임코드와 문맥을 기반으로 교육영상을 의미 단위 세그먼트로 자동 분할해야 함. | Must |
| FR-04 | 세그먼트 검토/수정 | - 운영자는 자동 생성된 세그먼트를 확인하고, 단건 수정·분할·병합·확정이 가능해야 함. | Should |
| FR-05 | 암묵적 신호 수집 | - 세그먼트 진입(segment_start), 완료(segment_complete), 되감기(seek_back), 일시정지(pause), 반복 시청(rewatch), 재생 속도 변경(speed_change), 다음 세그먼트 진입(next_segment_start), 영상 종료/이탈(exit) 등 주요 시청 행동 이벤트를 수집해야 함. | Must |
| FR-06 | 명시적 신호 수집 | - 최소 1종 이상의 직접 반응형 학습 신호를 수집해야 하며, MVP에서는 **이해도 체크(understood / unsure / review_again)**를 지원해야 함. | Must |
| FR-07 | 추가 명시적 신호 연동 | - 퀴즈 제출(quiz_submit), 핵심 개념 선택(concept_select), 한 줄 요약(summary_submit) 등 확장형 직접 반응 데이터를 연동할 수 있어야 함. | Should |
| FR-08 | 세그먼트 퍼널 계산 | - 각 세그먼트의 completion_rate, dropout_rate, rewatch_rate, next_transition_rate, explicit_response_rate, learning_stability_score를 계산해야 함. | Must |
| FR-09 | 학습 신호 통합 점수화 | - 명시적/암묵적 신호를 조합하여 세그먼트별 학습 안정성 또는 이해도 상태를 산출해야 함. | Should |
| FR-10 | 위험 구간 표시 | - 이탈 급증(dropout spike), 재시청 급증(rewatch spike), 이해도 저하(low explicit signal), 전환 감소(transition drop) 구간을 시각적으로 표시해야 함. | Must |
| FR-11 | 광고 위치 후보 산출 | - 세그먼트 경계/전환 구간 중 학습 흐름 방해 위험이 낮은 광고 후보 지점을 산출해야 함. | Must |
| FR-12 | Agent 추천 생성 | - Agent는 후보 지점 중 우선순위를 정하고, confidence score를 포함한 추천안을 생성해야 함. | Must |
| FR-13 | 추천 이유 제공 | - 추천마다 암묵적 신호 근거, 명시적 신호 근거, 제외 사유, 위험도, 기대효과를 구분하여 설명해야 함. | Must |
| FR-14 | 운영자 승인/수정 | - 운영자는 추천안을 승인(approve), 거절(reject), 수정(modify)할 수 있으며, 판단 메모(note)를 남길 수 있어야 함. | Must |
| FR-15 | 추천안 저장 | - 승인된 추천안은 recommendation_run 단위로 버전 관리되어 저장되어야 함. | Must |
| FR-16 | 비교 리포트 | - 추천 적용 전후 및 비교군 대비 CTR/completion/transition/explicit signal 성과를 조회할 수 있어야 함. | Should |
| FR-17 | 감사 로그 | - 누가 언제 어떤 추천을 생성·승인·수정·적용했는지 전체 이력을 기록해야 함. | Must |
| FR-18 | 결과 내보내기 | - 추천 결과를 CSV/API 형식으로 외부 운영 시스템에 전달할 수 있어야 함. | Could |

### (2) 구현 계획

**전체 시스템 아키텍처**

```text
[운영자 대시보드 - Next.js]
        ↓ HTTPS
      ALB (Application Load Balancer)
        ↓
[API 서버 - FastAPI / Uvicorn]
   ├─ PostgreSQL (운영 DB: 영상, 세그먼트, 추천, 성과, 감사 로그)
   ├─ Amazon S3 (영상 원본, 자막 파일, 리포트 파일 저장)
   ├─ RabbitMQ (메시지 브로커 - durable queue)
   └─ OpenTelemetry → Prometheus → Grafana (관측)
        ↓ Celery task dispatch
[비동기 Workers]
   ├─ video-processing queue  (영상/자막 전처리, 세그먼트 생성)
   ├─ metrics-aggregation queue  (raw events → segment_metric_snapshots 집계)
   └─ recommendation queue  (Agent 추천 후보 생성, 스코어링, LLM 설명 생성)

[비디오 플레이어 SDK] → POST /api/v1/player/events/batch → [API 서버] → [learning_events 테이블]
```

**데이터베이스 ERD 개요 (PostgreSQL, 20+ 테이블)**

```text
organizations
 └─ organization_members ─ users
 └─ videos
     ├─ video_assets (원본 파일 메타, S3 경로)
     ├─ caption_tracks → caption_cues (자막 트랙 및 개별 cue)
     ├─ video_signal_configs (영상별 신호 수집 설정)
     ├─ segment_sets (버전 관리: draft / finalized / archived)
     │   └─ segments (개별 세그먼트: start_ms, end_ms, title, topic, key_concepts)
     ├─ learner_sessions (익명 세션 관리)
     │   └─ learning_events (원시 이벤트: signal_mode=implicit/explicit/ad)
     ├─ segment_metric_snapshots (집계 지표: completion_rate, dropout_rate 등)
     ├─ recommendation_runs (추천 실행 단위)
     │   └─ recommendation_candidates (후보별 rank, confidence, slot 위치)
     │       ├─ recommendation_evidence (근거: implicit/explicit/exclusion)
     │       └─ recommendation_reviews (승인/반려/수정 이력)
     └─ ad_placements (최종 적용된 광고 위치)
         └─ placement_performance_daily (일별 광고 성과 집계)
 └─ guardrail_policies (가드레일 KPI 임계값 정의)
audit_logs (전체 감사 로그)
```

**MVP 마일스톤별 상세 구현 계획 (M1~M5, 총 43개 이슈)**

**M1. 프로젝트 골격 및 공통 기반 구축 (8개 이슈)**

- M1-02. Docker Compose 개발환경을 구성함.

  - 한 번의 `docker compose up` 명령으로 전체 스택을 기동할 수 있도록 함.
- M1-03. FastAPI 기본 앱을 구성함.

**M2. 교육영상 등록 및 세그먼트 생성 (9개 이슈)**

- M2-01. 영상 등록 API를 구현함.
  - `POST /api/v1/orgs/{orgId}/videos` 엔드포인트를 생성함.
  - 영상 파일 업로드(multipart/form-data) → S3 저장 → video_assets 메타데이터 기록을 처리함.
  - video_signal_configs 초기값을 자동 저장함.
- M2-02. 자막 트랙 및 cue를 저장함.
  - caption_tracks / caption_cues 테이블에 SRT/VTT 파싱 결과를 저장함.
  - 기본 자막 조회 API를 구현함.
- M2-03. 영상 상세/목록 API를 구현함.
  - 영상 목록(필터: 상태별/분석일), 영상 상세(메타데이터, 최근 분석 시각) 조회를 지원함.
- M2-04. 분석 작업 트리거를 구현함.
  - `POST /api/v1/orgs/{orgId}/videos/{videoId}/analyze` 엔드포인트를 구현함.
  - Celery video-processing queue에 job을 enqueue하고, 영상 상태를 uploaded → processing으로 변경함.
- M2-05. 세그먼트 자동 생성 워커를 구현함.
  - 자막 타임코드 기반 의미 단위 분절을 수행함.
  - 세그먼트별 title/topic/key_concepts를 NLP/LLM으로 자동 생성함.
  - segment_set(version=1, status=draft)과 segments를 DB에 저장함.
  - 분석 완료 시 영상 상태를 processing → analyzed로 변경함.
- M2-06. 세그먼트 세트 버전 관리를 구현함.
  - latest 세그먼트 세트 조회, clone(새 버전 생성), finalize(draft → finalized) 기능을 구현함.
- M2-07. 세그먼트 수정 API를 구현함.
  - 단건 수정(title, start_ms, end_ms 등), 분할(1개 → 2개), 병합(2개 → 1개) 기능을 구현함.
  - draft 상태에서만 수정 가능하도록 검증함.
- M2-08. 세그먼트 검토 화면을 구현함.
  - 세그먼트 목록(타임라인 표시), 세그먼트 상세 편집 UI, 분할/병합 버튼, 확정 액션을 구현함.
- M2-09. 감사 로그를 연결함.
  - 영상 생성, 분석 시작, 세그먼트 수정, finalize 이벤트를 audit_logs에 자동 기록함.

**M3. 학습 퍼널 수집·집계·시각화 (9개 이슈)**

- M3-01. learner session 생성/관리를 구현함.
  - 익명(anonymous) 세션 생성 규칙, player token 구조를 정의함.
  - 세션 종료 정책(30분 비활동 시 자동 종료 등)을 정의함.
- M3-02. 이벤트 수집 API를 구현함.
  - `POST /api/v1/player/events/batch` 엔드포인트를 구현함.
  - clientEventId 기반 멱등(idempotent) 처리로 중복 수집을 방지함.
  - signalMode(implicit/explicit/ad), eventType 필드를 검증함.
- M3-03. 이벤트 스키마/저장을 구현함.
  - learning_events 테이블에 원시 이벤트를 저장하고, payload를 jsonb로 보관함.
  - implicit / explicit / ad 이벤트를 signal_mode 컬럼으로 구분함.
- M3-04. 명시적 신호 구조를 구현함.
  - confidence_check_submit 이벤트를 지원함.
  - 응답 값: understood(이해함) / unsure(애매함) / review_again(다시 보기)을 처리함.
  - 세그먼트별 explicit_response_rate 계산을 준비함.
- M3-05. 집계 워커를 구현함.
  - Celery metrics-aggregation queue에서 raw events → segment_metric_snapshots로 집계함.
  - 집계 지표: completion_rate, dropout_rate, rewatch_rate, next_transition_rate, explicit_response_rate, confidence_understood_rate, confidence_unsure_rate, confidence_review_rate, learning_stability_score.
- M3-06. 위험 플래그 계산 로직을 구현함.
  - dropout spike (이전 세그먼트 대비 이탈률 급증) 탐지.
  - rewatch spike (비정상적 반복 시청 비율) 탐지.
  - low explicit signal (명시적 신호 응답률 저조) 탐지.
  - transition drop (다음 세그먼트 전환율 급감) 탐지.
- M3-07. 학습 퍼널 조회 API를 구현함.
  - `GET /api/v1/orgs/{orgId}/videos/{videoId}/funnel` 엔드포인트를 구현함.
  - 세그먼트별 메트릭 + risk flags를 함께 반환함.
- M3-08. 학습 퍼널 화면을 구현함.
  - 타임라인 기반 세그먼트 시각화, 세그먼트별 completion/dropout/rewatch 히트맵을 구현함.
  - 퍼널 테이블(세그먼트별 수치), explicit/implicit 필터 탭을 지원함.
  - 위험 구간을 빨간색 플래그/배지로 표시함.
- M3-09. 영상 개요 탭 KPI를 연동함.
  - 전체 완주율, 평균 다음 세그먼트 이동률, 이해도 체크 응답률을 영상 상세 개요 탭에 표시함.

**M4. Agent 추천 및 운영자 승인 플로우 (9개 이슈)**

- M4-01. 추천 입력 규격을 정의함.
  - 추천 엔진에 입력할 메트릭(completion_rate, dropout_rate, rewatch_rate, transition_rate, explicit signals)을 확정함.
  - 세그먼트 경계 후보 생성 기준, 가드레일 적용 순서를 정의함.
- M4-02. 추천 실행 API를 구현함.
  - `POST /api/v1/orgs/{orgId}/videos/{videoId}/recommendation-runs` 엔드포인트를 구현함.
  - Celery recommendation queue에 비동기 작업을 등록하고, recommendation_run 상태를 관리함(pending → running → completed / failed).
- M4-03. 추천 후보 생성 로직을 구현함.
  - 세그먼트 경계 기반 광고 삽입 후보(segment boundary candidate)를 자동 산출함.
  - 가드레일 정책 기반 제외 조건(예: completion_drop > 임계값인 구간)을 적용함.
  - 후보별 rank 및 confidence score를 산정함.
- M4-04. 근거 신호 생성 로직을 구현함.
  - implicit evidence (완주율, 이탈률, 재시청률 등 암묵적 지표 근거)를 생성함.
  - explicit evidence (이해도 응답률, understood 비율 등 명시적 지표 근거)를 생성함.
  - exclusion evidence (제외 사유: 가드레일 위반, 개념 설명 중간 등)를 생성함.
  - LLM을 활용하여 rationale summary(자연어 추천 설명)를 생성함.
- M4-05. 추천 후보 조회 API를 구현함.
  - recommendation_run 상세, candidate 목록, evidence 포함 응답을 반환함.
- M4-06. Agent 추천 화면을 구현함.
  - 추천 Top N 리스트, 추천 이유 카드(근거 지표 수치/위험도/confidence/기대효과)를 표시함.
  - 암묵적 신호 근거와 명시적 신호 근거를 구분 표시함.
  - 대안 슬롯 표시 및 타임라인 상 위치 표시를 구현함.
- M4-07. 추천 리뷰 액션 API를 구현함.
  - approve(승인), reject(반려), modify(위치 변경), note(운영 메모) 액션을 구현함.
  - recommendation_reviews 테이블에 이력을 기록함.
- M4-08. 추천 적용 API를 구현함.
  - 승인/수정된 candidate → ad_placements 레코드를 생성함.
  - 적용 상태(applied)를 반영하고, 감사 로그를 기록함.
- M4-09. 추천 관리 목록 화면을 구현함.
  - 대기/승인/반려 필터, 위험도 필터, 영상별 검토 진입을 지원함.

**M5. 성과 리포트, 가드레일, 안정화 (8개 이슈)**

- M5-01. placement 성과 집계 로직을 구현함.
  - ad_impression / ad_click 이벤트를 placement에 연결하고, placement_performance_daily로 일별 집계함.
- M5-02. 영상 성과 비교 API를 구현함.
  - CTR 전후 비교, completion_rate / transition_rate / explicit_signal 전후 비교를 반환함.

**기술적 기대효과**

SegmentFlow – 교육영상 세그먼트 기반 학습 퍼널 분석 및 Agent 광고 위치 추천 시스템

### (2) 프로젝트 목적

- 교육영상을 자막/문맥 기반으로 **세그먼트(의미 단위 구간)로 자동 분할**하고, 운영자가 이를 검토·수정할 수 있도록 함.
- 세그먼트별로 **학습 퍼널(진입 → 완료 → 이탈 → 재시청 → 다음 세그먼트 전환 → 이해도 확인)**을 분석하여, 영상 전체가 아닌 구간 단위로 학습 흐름을 파악함.
- **암묵적 학습 신호**(완료율, 이탈률, 되감기, 일시정지, 배속 변화 등)와 **명시적 학습 신호**(이해도 체크 등 직접 반응)를 통합 수집·분석하여, 퀴즈 유무에 관계없이 학습 안정성을 판단함.
- 분석 결과를 기반으로 **AI Agent가 광고 위치 후보를 산출**하고, 각 후보에 대해 **추천 근거(근거 지표, 위험도, 기대효과)**를 설명 가능하게 제시함.
- 최종 광고 위치 결정은 자동 집행이 아닌 **운영자의 승인·수정·반려(Human-in-the-loop)**를 거치도록 설계함.
- 추천 적용 후 광고 성과(CTR)와 **학습 가드레일 KPI(완주율 하락폭, 전환율 변화 등)**를 동시 모니터링함.

## 2. 과제 수행 과정

### (1) 설계 및 계획

- PRD(제품 요구사항 정의서)를 작성하여 제품 비전, 5대 핵심 문제, 사용자 페르소나(운영자/콘텐츠 기획자/광고 매니저), End-to-End 사용자 여정, 18개 기능 요구사항, 비기능 요구사항을 확정함.
- DB 스키마 문서를 작성하여 5대 설계 원칙(세그먼트 버전 관리, 신호 구분 저장, 원시/집계 분리, 추천 버전 관리, 승인 이력 별도 저장)과 20+ 테이블 ERD를 확정함.
- API 명세서를 작성하여 Base URL, 인증 방식(JWT/Player Token), 멀티테넌시 규칙, 공통 응답 포맷, 핵심 엔드포인트를 정의함.
- IA/와이어프레임 문서를 작성하여 전체 화면 구조(대시보드, 영상 관리, 추천 관리, 리포트, 설정), 핵심 화면 와이어프레임(로그인, 대시보드, 영상 목록, 영상 상세 6개 탭, 추천 화면, 리포트)을 확정함.
- 기술 스택 문서를 작성하여 4대 선정 원칙(MVP 속도, 비동기 대응력, 대시보드 생산성, 안정성)에 따른 10개 영역별 스택과 선정 사유를 확정함.
- MVP 마일스톤 문서를 작성하여 5개 마일스톤(M1~M5), 43개 핵심 이슈, 완료 기준, 의존성, 리스크를 정의함.
- 시스템 아키텍처를 설계함: Next.js 대시보드 → ALB → FastAPI API → Celery Workers(video-processing / metrics-aggregation / recommendation) → PostgreSQL + S3 + RabbitMQ.

### (2) 구현 내용

**M1. 프로젝트 골격 및 공통 기반 구축**

- 모노레포 구조(`apps/web`, `apps/api`, `apps/worker`, `packages/*`)를 생성하고 pnpm workspace를 구성함.
- Docker Compose 개발환경(postgres, rabbitmq, api, worker, web)을 구성하여 한 번의 명령으로 전체 스택을 기동할 수 있도록 함.
- FastAPI 기본 앱(health check, version endpoint, 공통 exception handler, request id middleware, OpenAPI 문서)을 구성함.
- Next.js 기본 앱(로그인 화면, 좌측 사이드바, 상단 글로벌 바, 빈 대시보드 shell)을 구현함.
- JWT 인증, organization context, admin/operator/analyst role guard를 구현함.
- Alembic 마이그레이션 체계를 구축하고 초기 핵심 테이블(organizations, users, videos, guardrail_policies, audit_logs)을 생성함.
- OpenTelemetry + Prometheus + Grafana 로컬 관측 환경을 구성하고, CI 파이프라인(lint, type check, test, build)을 자동화함.

**M2. 교육영상 등록 및 세그먼트 생성**

- 영상 등록 API(`POST /videos`)를 구현하여 파일 업로드 → S3 저장 → video_assets 메타 기록 → signal config 초기값 저장을 처리함.
- 자막 파싱(SRT/VTT) → caption_tracks / caption_cues 저장, 자막 조회 API를 구현함.
- 분석 작업 트리거(`POST /videos/{videoId}/analyze`)를 구현하여 Celery job enqueue 및 상태 변경(uploaded → processing)을 수행함.
- 세그먼트 자동 생성 워커를 구현하여 자막 타임코드 기반 의미 단위 분절, NLP/LLM 기반 title/topic/key_concepts 자동 생성, segment_set(draft) → segments 저장을 수행함.
- 세그먼트 세트 버전 관리(latest 조회, clone, finalize)를 구현함.
- 세그먼트 수정 API(단건 수정, 분할, 병합, draft 상태 검증)를 구현함.
- 세그먼트 검토 화면(타임라인, 상세 편집, 분할/병합 버튼, 확정 액션)을 구현함.
- 감사 로그 연결: 영상 생성, 분석 시작, 세그먼트 수정, finalize 이력을 자동 기록함.

**M3. 학습 퍼널 수집·집계·시각화**

- 이벤트 수집 API(`POST /player/events/batch`)를 구현하여 clientEventId 멱등 처리, signalMode/eventType 검증을 수행함.
- 암묵적 신호(segment_start, segment_complete, pause, seek_back, rewatch, speed_change, next_segment_start, exit)를 learning_events에 저장함.
- 명시적 신호(confidence_check_submit: understood/unsure/review_again)를 learning_events에 signal_mode=explicit으로 분리 저장함.
- 집계 워커를 구현하여 raw events → segment_metric_snapshots(completion_rate, dropout_rate, rewatch_rate, next_transition_rate, explicit_response_rate, confidence_* rate, learning_stability_score)를 집계함.
- 위험 플래그 계산 로직(dropout spike, rewatch spike, low explicit signal, transition drop)을 자동 탐지하도록 구현함.
- 학습 퍼널 화면(타임라인, 세그먼트 히트맵, 퍼널 테이블, explicit/implicit 필터 탭, 위험 구간 빨간 배지)을 구현함.
- 영상 개요 탭에 전체 완주율, 평균 전환율, 이해도 응답률 KPI를 연동함.

**M4. Agent 추천 및 운영자 승인 플로우**

- 추천 실행 API(`POST /videos/{videoId}/recommendation-runs`)를 구현하여 Celery recommendation queue에 비동기 작업을 등록하고 recommendation_run 상태를 관리함.
- 추천 후보 생성 로직을 구현함: 세그먼트 경계 후보 산출 → 암묵적 신호 기반 학습 흐름 안정성 점검 → 명시적 신호 기반 이해도 저하 보정 → 가드레일 정책 필터링 → rank/confidence 산정.
- 근거 신호 생성 로직(implicit evidence, explicit evidence, exclusion evidence)을 구현하고, LLM을 활용하여 rationale summary를 생성함.
- Agent 추천 화면(Top N 리스트, 추천 이유 카드, 암묵적/명시적 근거 구분 표시, 위험도, confidence, 타임라인 위치)을 구현함.
- 추천 리뷰 액션(approve/reject/modify/note) API를 구현하여 recommendation_reviews에 이력을 기록함.
- 추천 적용 API를 구현하여 승인/수정된 candidate → ad_placements 레코드를 생성함.
- 추천 관리 목록 화면(대기/승인/반려 필터, 위험도 필터)을 구현함.

**M5. 성과 리포트, 가드레일, 안정화**

- ad_impression/ad_click 이벤트를 placement에 연결하고, placement_performance_daily로 일별 집계를 수행함.
- 영상 성과 비교 API(CTR 전후 비교, completion/transition/explicit signal 전후 비교, candidate별 결과)를 구현함.
- 가드레일 판정 로직(completion_drop, transition_drop, explicit_signal_drop threshold 기반 pass/warning/violation)을 구현함.
- 성과 리포트 화면(적용 전/후 비교 차트, 가드레일 상태 배지, candidate별 성과 카드)을 구현함.
- 감사 로그 화면(추천 생성/승인/수정/적용/세그먼트 변경 이력 조회 UI)을 구현함.
- 시스템 안정화(API 응답 속도 점검, worker retry 정책, queue backlog 모니터링, DB 인덱스 보강)를 수행함.
- E2E 테스트(핵심 happy path 자동화: 영상 등록→분석→세그먼트→퍼널→추천→승인→리포트)를 수행함.
- 파일럿 운영 체크리스트(운영 가이드, 이벤트 수집 가이드, 장애 대응 가이드, KPI 측정 기준)를 작성함.

### (2) 기대효과 및 향후 계획

**기대효과**

- 교육영상 운영 의사결정 고도화: 세그먼트 단위 분석과 AI Agent 추천을 통해, 운영자가 **"어디에 광고를 넣어야 학습을 보존하면서 성과를 낼 수 있는가"**를 데이터 기반으로 판단할 수 있는 운영 도구 프로토타입을 확보함.
- 다중 학습 신호 통합 분석 프레임워크 검증: 암묵적 신호(시청 행동)와 명시적 신호(이해도 체크)를 통합 해석하는 접근이 퀴즈 단일 신호보다 학습 상태를 더 풍부하게 파악할 수 있음을 확인함.
- Human-in-the-loop AI 추천 시스템 실현: 완전 자동화가 아닌 설명 가능한 추천 + 사람 승인 구조가 교육 도메인에서 운영자 신뢰를 확보하는 데 효과적임을 검증함.
- 풀스택 프로덕션급 시스템 개발 경험: Frontend(Next.js), Backend(FastAPI), 비동기 파이프라인(Celery/RabbitMQ), AI(LLM Agent), 인프라(Docker/AWS)를 아우르는 실무 수준 개발 역량을 축적함.
